1 Difference between polymorphism and metamorphism⌗

This chapter will explain the difference between polymorphism and metamorphism within malware.

1.1 Polymorphism⌗

Polymorphism refers to the alteration of distinguishable attributes with the intention of avoiding detection. Programs possess various identifiable attributes that are employed by researchers and antivirus companies to recognize them. These attributes encompass, but are not restricted to, filenames, file types, encryption keys, or strings contained within the binary.

The purpose of employing polymorphism is to elude pattern-matching detection mechanisms utilized by security solutions such as antivirus software or endpoint detection and response (EDR) solutions. By modifying only the identifiable attributes, polymorphism does not alter the core functionality of the malware. Consequently, when a security solution generates a signature for a polymorphic malware, the malware simply needs to make a change in order to evade detection.

1.2 Metamorphism⌗

Metamorphism is characterized by the rewriting of malware at each iteration, without relying on an encryption key. It serves as the more menacing sibling of polymorphism. With each binary iteration, the malware evolves into a more sophisticated version, while maintaining its core functionality. Unlike polymorphism, metamorphism does not alter the identifiable features of the binary, but instead modifies the appearance of the code itself. Through self-translation and rewriting, metamorphic malware achieves a significant transformation, making two iterations appear entirely distinct from one another.
